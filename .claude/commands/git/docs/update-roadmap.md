`docs/ROADMAP.md` 의 완료된 작업을 점검하고 체크해줘.

대상: `$ARGUMENTS` 가 특정 태스크 ID(예: `T1.7`)면 그 태스크만, **비어 있으면 아직 ✅ 안 된 모든 태스크**를 점검한다.

## 핵심 원칙 — 추측 금지, 증거 기반 (이게 제일 중요)

대화 맥락이나 "했다고 들음" 으로 체크하지 말 것. 각 태스크를 완료로 표시하기 **전에 반드시 실제 프로젝트 상태로 검증**한다:

- **산출물 존재**: 그 태스크의 "산출물/세부 단계" 에 적힌 파일이 실제로 있는지 Read/Glob 로 확인.
- **테스트 통과**: 그 태스크 "테스트 계획" 의 명령을 직접 실행 — 예: `npm run test:quotes`(데이터 레이어), Playwright MCP(UI/페이지 흐름).
- **게이트 통과**: 해당되면 `npm run build`(Cache Components 게이트), `npm run lint`, `npx tsc --noEmit` 를 실제로 돌려 결과를 본다.
- **DoD 대조**: 그 태스크/Phase 의 "완료 정의(DoD)" 항목을 하나씩 대조한다.

→ DoD 가 **하나라도 미충족이면 체크하지 말고**, 무엇이 빠졌는지 보고한다. (잘못 ✅ 하는 것보다 ⬜ 로 남기는 게 낫다.)

## 절차

1. `docs/ROADMAP.md` 를 읽어 태스크 목록·각 태스크 DoD·Phase 완료 정의 체크리스트를 파악한다.
2. 대상 태스크마다 위 "핵심 원칙" 의 검증을 **실제로 실행**한다(파일은 Read, 명령은 Bash/MCP 로 직접). 시간 절약을 위해 점검 대상과 무관한 명령은 돌리지 않는다.
3. 검증을 통과한 것만 `docs/ROADMAP.md` 에 반영한다:
   - 작업 항목 헤더(`#### T1.x — ...`)에 완료 표식(✅) 추가/갱신.
   - Phase "완료 정의(DoD)" 의 체크박스 `- [ ]` → `- [x]` (실제 충족된 항목만).
   - "변경 이력" 표에 한 줄 추가: 일자=오늘(절대일자), 변경 요약, 작성자=`/git:docs:update-roadmap`.
4. `CLAUDE.md` 의 "진행 상태 / 일정" 절이 ROADMAP 과 어긋나면 같이 맞춘다(완료 태스크 목록 + 다음 차례 태스크).
5. **문서만 수정한다 — 커밋은 하지 않는다.** 커밋이 필요하면 보고 끝에 `/git:commit` 사용을 안내한다.

## 보고 (한국어)

- ✅ **새로 완료 처리한 태스크** — 각각 **무엇으로 확인했는지 근거**(파일 경로 / 테스트 출력 / 빌드 결과)를 한 줄씩.
- ⬜ **아직 미완료** — 빠진 DoD 항목을 구체적으로.
- ▶️ **다음 크리티컬 패스 태스크** 1개 + 권장 담당 에이전트(`quote-viewer-builder` 로직·라우팅 / `quote-ui-designer` UI·디자인).
