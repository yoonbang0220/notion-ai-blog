이미 구현·배포된 MVP PRD를 고도화할 때는 **본문을 덮어쓰지 않고** 문서 끝에 "부록 A. vN.x 고도화" 섹션을 통째로 추가한다. 본문은 SSOT 이력이자 구현된 사실이므로 보존.

**Why:** MVP 본문은 이미 구현된 코드와 1:1로 대응하는 기록이다. 덮어쓰면 "무엇이 MVP였고 무엇이 추가인지" 추적 불가. 고도화가 MVP 가정을 뒤집을 때 특히 둘을 구분해 보여줘야 한다.

**How to apply:**
- 문서 상단에 "문서 구성 안내"로 본문=MVP / 부록=고도화 구분 명시.
- 고도화가 MVP의 기존 가정/주석을 뒤집으면, 본문 해당 줄은 **그대로 두고** 바로 아래 `〔v1.x 변경〕 …(부록 A 참조)` 인라인 각주만 추가. (예: "운영자 화면 안 만든다" → 부록에서 도입됨 표시)
- 부록은 Overview/명세서/IA/Wireframe/기획안/DataTable/결정사항/자가검증을 **고도화분만** 미니 PRD로 자기완결.
- 이미 구현된 기능(예: 다크모드 next-themes)은 "신규 개발"로 적지 말고 **"이미 구현됨 + 검증·문서화/선택적 폴리시"**로 격하. 과설계 금지.
- 고도화 DataTable은 "기존 타입에 이미 있는 필드 / 신규 필드"를 출처 열로 구분 — 실측으로 신규 Notion 속성 0건임을 확인 후 기록.
- MVP의 Future 목록은 건드리지 않는다. 새 고도화가 Future 항목과 혼동되지 않게 명시적으로 구분(예: "관리자 목록"≠"통계 대시보드").
