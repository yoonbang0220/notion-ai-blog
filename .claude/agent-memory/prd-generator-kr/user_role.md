사용자(berdanis4@gmail.com)는 이 저장소에서 Notion 기반 MVP들을 **혼자 기획하고 본인이 직접 구현**한다. 즉 PRD의 1차 독자는 사용자 본인(+ Claude Code 에이전트들)이며, 외부 개발자에게 발주하는 산출물이 아니다.

**Why:** PRD를 받자마자 `prd-roadmap-architect` → 코드 작성 흐름으로 이어진다. 그래서 PRD는 (1) 본인이 일주일 뒤 다시 읽어도 의도를 잃지 않을 만큼 명세가 분명하고 (2) 코드 에이전트(특히 Claude Code)가 그대로 구현 태스크로 분해할 수 있을 만큼 정합성 규칙·기술 결정이 박혀 있어야 한다. 외부 발주용 PRD처럼 영업·시장조사·KPI 그래프 같은 장식 절은 필요 없다.

**How to apply:**
- PRD 내 [정합성 규칙]·[처리 프로세스]·[반드시 해야할 기술 스택] 절을 두텁게 쓴다(에이전트가 구현 시 그대로 가져다 쓰는 절).
- "어떤 가정을 했는가 / 운영자가 결정해줘야 할 열린 질문" 절을 PRD 끝에 항상 둔다 — 본인이 의사결정 누락 없이 다음 단계로 넘어갈 수 있게.
- 한국어 본문 + 영문 변수/컬럼명 혼용 그대로. 사용자가 코드 작성 시 그대로 복사해 쓰는 일이 잦다.
- 톤은 "본인 + 본인을 돕는 에이전트" 가 읽는다고 가정한 실무 톤. 영업적 수식·이모지 배제.
