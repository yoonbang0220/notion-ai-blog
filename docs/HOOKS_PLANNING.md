# Claude Code Hooks + Slack 알림 시스템 기획서

> 작성일: 2026-05-15 | 버전: 1.0 | 작성자: 역기획 (구현 완료 후 문서화)

---

## 1. 개요

Claude Code(AI 코드 작성 도구)가 작업을 수행하는 동안 개발자가 자리를 비우거나 다른 업무를 하고 있어도, Slack의 `#hooks-notification` 채널로 실시간 알림을 받아 AI 에이전트의 상태를 즉시 파악할 수 있도록 하는 자동화 알림 시스템이다. Hook(훅, AI 도구가 특정 시점에 자동으로 실행하는 연결 장치)을 활용해 별도의 조작 없이 5가지 핵심 이벤트를 Slack 메시지로 변환한다.

---

## 2. 배경 & 문제 정의

### Claude Code 사용 시 발생하는 불편함

Claude Code는 코드 작성, 파일 수정, 명령 실행 등 복잡한 작업을 자율적으로 수행한다. 특히 서브에이전트(하위 AI 작업 단위)를 활용한 멀티에이전트 작업에서는 작업 시간이 수 분에서 수십 분까지 길어질 수 있다.

| 상황 | 문제 |
|------|------|
| 긴 작업 실행 중 자리를 비운 경우 | 작업이 완료됐는지 알 수 없어 화면을 반복 확인해야 함 |
| 권한 요청(Permission Request)이 발생한 경우 | 사용자 응답을 기다리며 작업이 멈춰있는 줄 모름 |
| 서브에이전트가 여러 개 병렬로 실행되는 경우 | 어느 에이전트가 무슨 작업을 하고 있는지 파악 불가 |
| 여러 프로젝트를 동시에 진행하는 경우 | 어떤 프로젝트에서 이벤트가 발생했는지 구분 불가 |

### 기존 방법의 한계

Claude Code는 터미널(명령줄 화면) 내에서만 상태를 표시한다. 터미널 창을 닫거나 다른 앱으로 전환하면 작업 상태를 알 방법이 없다. 별도의 알림 연동 기능도 기본 제공되지 않는다.

---

## 3. 목표 & 비목표

### 목표 (이 시스템이 하는 일)

- Claude Code의 5가지 핵심 이벤트를 Slack 알림으로 자동 전달한다.
- 알림 메시지에 프로젝트명, 상태, 시간(KST 기준)을 통일된 형식으로 포함한다.
- macOS와 Windows 환경 모두에서 동작한다.
- 서브에이전트 작업 중임을 감지해, 메인 에이전트의 도구 사용과 서브에이전트의 도구 사용을 구분한다(불필요한 중복 알림 차단).
- 개인 Webhook URL(민감 정보)이 Git 저장소에 올라가지 않도록 보호한다.

### 비목표 (이 시스템이 하지 않는 일)

- Claude Code의 모든 도구 사용을 알림으로 보내지는 않는다(메인 에이전트의 도구 사용은 의도적으로 제외).
- Slack 외 다른 채널(이메일, 카카오톡 등)에는 보내지 않는다.
- Claude Code의 실행을 제어하거나 작업 결과를 변경하지 않는다(알림 전용 시스템).
- Rate Limit(알림 발송 횟수 제한) 처리는 현재 포함되지 않는다.

## 4. 사용자 시나리오

### 시나리오 A — 장시간 작업 중 자리를 비운 개발자

**페르소나:** 윤방 (프론트엔드 개발자, 1인 개발, Claude Code를 자주 사용)

**상황:**
윤방은 대형 리팩토링(코드 구조 개선) 작업을 Claude Code에 맡기고 커피를 마시러 자리를 떴다. 작업이 30분 정도 걸릴 것으로 예상하고, 화면을 계속 보고 있을 수 없는 상황이다.

**행동 흐름:**
1. Claude Code가 작업을 시작하고 서브에이전트를 생성한다.
2. 윤방의 Slack에 `🚀 서브에이전트 시작` 알림이 온다 — 에이전트 이름과 작업 내용이 포함돼 있다.
3. 작업 중 파일 수정, 명령 실행 등이 있을 때마다 `🔄 서브에이전트 실행 중` 알림이 온다.
4. 윤방은 스마트폰으로 Slack을 보며 작업이 진행 중임을 확인한다.
5. 작업이 끝나면 `✅ 작업 완료!` 알림이 온다 — 윤방은 즉시 자리로 돌아가 결과를 확인한다.

**효과:** 화면을 반복적으로 확인하지 않아도 되며, 작업 완료 즉시 대응할 수 있다.

---

### 시나리오 B — 권한 요청으로 작업이 멈춘 상황

**페르소나:** 동일 개발자, 다른 상황

**상황:**
Claude Code가 시스템 파일을 삭제하려는 명령을 실행하려 했고, 이에 대한 사용자 승인을 요청하며 작업이 일시 중단됐다.

**행동 흐름:**
1. Claude Code가 권한 확인이 필요한 명령에 도달한다.
2. 윤방의 Slack에 `🔒 권한 요청` 알림이 즉시 온다 — 어떤 도구를, 어떤 명령으로 실행하려 했는지 상세가 포함된다.
3. 윤방은 내용을 보고 위험 여부를 판단해 터미널로 돌아가 승인 또는 거부한다.
4. 작업이 재개되고 완료 시 `✅ 작업 완료!` 알림이 온다.

**효과:** 승인 대기 상태가 방치되지 않으며, 보안 위험 행동을 즉시 인지할 수 있다.

## 5. 아키텍처 개요

### 데이터 흐름

```
Claude Code 실행
    │
    ├─[이벤트 발생]──→ Claude Code 내부 Hook 트리거
    │                      │
    │                      ├─ stdin(표준 입력, 프로그램에 데이터를 넘기는 통로)으로 이벤트 JSON 전달
    │                      └─ 인수(argument, 프로그램 실행 시 전달하는 값)로 이벤트 타입 전달
    │
    ├─[macOS]──→ slack-notify.sh (bash + python3)
    │                      │
    │                      ├─ JSON 파싱 → 메시지 포맷 생성
    │                      └─ curl로 Slack Webhook URL에 POST 전송
    │
    └─[Windows]──→ slack-notify.ps1 (PowerShell)
                           │
                           ├─ JSON 파싱 → 메시지 포맷 생성
                           └─ curl.exe로 Slack Webhook URL에 POST 전송
                                      │
                                      └─→ Slack #hooks-notification 채널에 메시지 표시
```

### 마커 파일 패턴 (서브에이전트 감지의 핵심)

Claude Code의 Hook 중 `PostToolUse`(도구 사용 직후 실행되는 훅)는 메인 에이전트와 서브에이전트 모두에서 동일하게 발동된다. 이를 구분하지 않으면 메인 에이전트의 모든 도구 사용(파일 읽기, 명령 실행 등)에도 알림이 전송되어 Slack이 수십 개의 불필요한 메시지로 가득 찬다.

이를 해결하기 위해 **마커 파일 방식**을 도입했다.

```
SubagentStart 훅 발동
    └─→ /tmp/claude-subagent-active.txt 파일 생성
            └─ 파일 내용: 에이전트 이름 기록

PostToolUse 훅 발동 (서브에이전트 실행 중)
    └─→ 마커 파일 존재 확인
            ├─ [파일 있음] → Slack 알림 전송 (서브에이전트 실행 중)
            └─ [파일 없음] → 조용히 종료, 알림 없음 (메인 에이전트 도구 사용)

SubagentStop 훅 발동
    └─→ /tmp/claude-subagent-active.txt 파일 삭제
```

macOS에서는 `/tmp/claude-subagent-active.txt`, Windows에서는 `%TEMP%\claude-subagent-active.txt` 경로를 사용한다.

### 설정 파일 구조

| 파일 | 적용 범위 | 역할 |
|------|-----------|------|
| `/Users/yoonbang/.claude/settings.json` | 전역(모든 프로젝트) | Hook 5개 + Slack Webhook URL 환경 변수 등록 |
| `/Users/yoonbang/nextjs-shadcn-starter/.claude/settings.json` | 이 프로젝트 한정 | PreToolUse 테스트 훅 (개발 검증용) |
| `.claude/settings.local.json` | 로컬 전용 | 개인 Webhook URL 보관용 (Git 제외) |

## 6. 상세 기능 — 5가지 이벤트

### 6-1. 권한 요청 (permission)

**발동 시점:** Claude Code가 사용자 승인이 필요한 도구(위험하거나 민감한 명령)를 실행하려 할 때

**Hook 종류:** `PermissionRequest`

**Slack 메시지 예시:**

```
🔒 *권한 요청* [14:32:11]
*🔧 도구:* `Bash`
*📝 내용:* `rm -rf ./dist && rm -rf ./node_modules`
*📁 프로젝트:* `nextjs-shadcn-starter`
*📊 상태:* 권한 요청 대기 중
*🕐 시간:* 14:32:11 (KST)
```

**포함 정보:** 도구명, 실행하려는 명령어 또는 파일 경로(최대 80자), 프로젝트명, 시간

---

### 6-2. 작업 완료 (stop)

**발동 시점:** Claude Code가 응답을 마치고 사용자의 다음 입력을 기다리는 상태가 될 때

**Hook 종류:** `Stop`

**Slack 메시지 예시:**

```
✅ *작업 완료!* [15:04:22]
*📁 프로젝트:* `nextjs-shadcn-starter`
*📊 상태:* 작업 완료
*🕐 시간:* 15:04:22 (KST)
```

**포함 정보:** 프로젝트명, 시간 (작업 완료 자체가 핵심이므로 별도 상세 없음)

---

### 6-3. 서브에이전트 시작 (subagent_start)

**발동 시점:** Claude Code가 하위 AI 에이전트(SubAgent)를 새로 생성해 독립적인 작업을 시작시킬 때

**Hook 종류:** `SubagentStart`

**부가 동작:** 마커 파일 생성 (에이전트 이름 기록)

**Slack 메시지 예시:**

```
🚀 *서브에이전트 시작* [14:55:03]
*🤖 에이전트:* `code-reviewer`
*📋 작업:* TypeScript 타입 오류 검사 및 수정 작업을 수행합니다...
*📁 프로젝트:* `nextjs-shadcn-starter`
*📊 상태:* 서브에이전트 작업 시작
*🕐 시간:* 14:55:03 (KST)
```

**포함 정보:** 에이전트 이름, 작업 설명(최대 100자), 프로젝트명, 시간

**에이전트 이름 추출 우선순위:** `subagent_type` → `agent_type` → `agent_name` → `name` → `type` 순서로 시도, 모두 없으면 "Unknown"

---

### 6-4. 서브에이전트 실행 중 (subagent_running)

**발동 시점:** 서브에이전트가 개별 도구(파일 읽기, 명령 실행 등)를 사용한 직후

**Hook 종류:** `PostToolUse` + 마커 파일 존재 여부 필터

**대상 도구:** `Bash`, `Write`, `Edit`, `Read` (settings에서 matcher로 지정)

**Slack 메시지 예시:**

```
🔄 *서브에이전트 실행 중* [14:55:47]
*🤖 에이전트:* `code-reviewer`
*🔧 도구:* `Edit`
*📝 내용:* `app/components/Header.tsx`
*📁 프로젝트:* `nextjs-shadcn-starter`
*📊 상태:* 서브에이전트 실행 중
*🕐 시간:* 14:55:47 (KST)
```

**포함 정보:** 마커 파일에서 읽은 에이전트 이름, 사용한 도구명, 작업 상세(명령어 최대 60자, 또는 파일 경로), 프로젝트명, 시간

---

### 6-5. 서브에이전트 완료 (subagent_stop)

**발동 시점:** 서브에이전트가 작업을 마치고 종료될 때

**Hook 종류:** `SubagentStop`

**부가 동작:** 마커 파일 삭제

**Slack 메시지 예시:**

```
🎯 *서브에이전트 완료* [15:02:58]
*🤖 에이전트:* `code-reviewer`
*📋 작업:* TypeScript 타입 오류 검사 및 수정 작업을 수행합니다...
*📁 프로젝트:* `nextjs-shadcn-starter`
*📊 상태:* 서브에이전트 작업 완료
*🕐 시간:* 15:02:58 (KST)
```

**포함 정보:** 에이전트 이름, 작업 설명, 프로젝트명, 시간

## 7. 기술 스택 & 호환성

### 구성 요소

| 구성 요소 | 기술 | 역할 |
|-----------|------|------|
| Hook 실행 주체 | Claude Code 내부 엔진 | 이벤트 발생 시 스크립트 자동 호출 |
| macOS 알림 스크립트 | bash + python3 | JSON 파싱 및 Slack 메시지 전송 |
| Windows 알림 스크립트 | PowerShell 5.1+ | JSON 파싱 및 Slack 메시지 전송 |
| JSON 전송 도구 | curl (macOS 내장) / curl.exe (Windows) | Slack Webhook API(외부 서비스와 데이터를 주고받는 연결통로)에 HTTP POST 요청 |
| Slack 수신 | Slack Incoming Webhook | `#hooks-notification` 채널에 메시지 표시 |
| 마커 파일 | /tmp 또는 %TEMP% 임시 디렉터리 | 서브에이전트 활성 상태 임시 저장 |

### 플랫폼별 설계 차이

**macOS (bash + python3):**
- python3를 임시 파일로 분리해 실행 — stdin(표준 입력)을 Hook의 JSON 데이터 전용으로 온전히 보존
- heredoc(여러 줄 문자열을 스크립트에 내장하는 방식)으로 python 코드를 임시 파일에 저장 후 실행
- curl 기본 내장 활용

**Windows (PowerShell):**
- `ConvertFrom-Json`으로 JSON 파싱
- PowerShell 5.1에서 이모지(서로게이트 페어) 잘림 방지를 위해 임시 파일을 UTF-8 BOM 없이 저장 후 curl.exe로 전송
- 마커 파일 경로를 `$env:TEMP` 환경 변수로 참조

### 보안 고려 사항

- Slack Webhook URL은 환경 변수(`SLACK_WEBHOOK_URL`)로만 참조하며, 스크립트 코드에 직접 포함하지 않는다.
- 전역 설정 파일(`~/.claude/settings.json`)에는 `env` 블록으로 주입하며, 이 파일은 홈 디렉터리에만 존재해 프로젝트 Git 저장소에 올라가지 않는다.
- 프로젝트 내 개인 설정이 필요한 경우 `.claude/settings.local.json`을 사용하며 `.gitignore`에 명시해 저장소 제외를 강제한다.

## 8. 알려진 이슈 & 한계

### 수정 완료된 버그

**BUG-001 (High, 수정 완료)**
- **현상:** 이벤트 JSON의 `description` 필드가 숫자형(정수, 실수, 참/거짓 값)으로 올 때 스크립트가 충돌하며 종료됨
- **원인:** python의 `len()` 함수는 문자열에만 사용 가능한데, 숫자 타입에 직접 호출
- **수정 방법:** `description`을 `str()` 함수로 문자열 변환 후 길이 체크

### 미수정 버그

**BUG-002 (Medium, 보류)**
- **현상:** `tool_input` 필드가 JSON 객체가 아닌 JSON 문자열로 전달될 때 내부 필드(command, file_path 등) 파싱 실패
- **영향:** 도구 상세 정보가 Slack 메시지에 표시되지 않음 (알림은 정상 전송)
- **임시 대응:** 없음. 알림 자체는 전송되나 상세 정보 일부가 빠질 수 있음

**BUG-003 (Low, 보류)**
- **현상:** `get_first()` 헬퍼 함수가 값이 `0`, `false`, 빈 문자열인 경우를 "값 없음"으로 처리
- **영향:** 에이전트 이름이나 설명이 이런 falsy 값(거짓으로 판단되는 값)인 경우 누락
- **임시 대응:** 없음. 실제 사용 사례에서 발생 빈도 낮음

### 구조적 한계

| 한계 | 설명 | 개선 방향 |
|------|------|-----------|
| Rate Limit 부재 | 도구 사용이 빈번한 서브에이전트에서 단시간에 수십 개 알림 발송 가능 | 쿨다운(최소 전송 간격) 로직 추가 |
| 단일 에이전트 마커 | 마커 파일이 하나뿐이어서 서브에이전트가 동시에 두 개 이상 실행 시 이름 추적 불정확 | 마커 파일을 PID(프로세스 식별 번호) 기반으로 분리 |
| Windows 미완성 | Windows의 `~/.claude/settings.json`에 PostToolUse 훅은 사용자가 수동으로 추가해야 함 | Windows 설정 자동화 스크립트 제공 |
| PowerShell 이모지 잘림 위험 | `.Substring()` 메서드가 이모지 같은 4바이트 유니코드 문자를 2바이트 단위로 자를 수 있음 | 문자 단위 Substring 구현으로 교체 |
