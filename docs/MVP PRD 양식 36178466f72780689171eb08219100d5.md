# 참고 사항

## MVP 원칙

- MVP는 말 그대로 MVP다. 풀스펙으로 기획하거나 개발하지 않는다.
- 핵심 기능이 유저들에게 전달되고, 주고 싶은 경험(Goal)에 도달하는지 빠르게 파악하는 것이 목적이다.
- 따라서 핵심 기능 위주로만 기획하고 개발하며, 부가 기능은 검증 이후로 미룬다.
- **판단 기준:** "이 기능이 없으면 Goal을 검증할 수 없는가?" → Yes면 포함, No면 제외
- **범위 경계:** Output의 Wire-frame, IA, 세부 기획은 핵심 기능에 한정해서 작성한다.
