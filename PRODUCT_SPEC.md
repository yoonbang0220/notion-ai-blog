# StarterKit 역기획서 (Reverse Planning Document)

> 작성일: 2026-05-14 | 버전: 1.0 | 분석 대상: nextjs-shadcn-starter

---

## 1. 프로젝트 개요

### 서비스 목적

StarterKit은 웹 서비스를 새로 만들 때 매번 반복해야 하는 기본 설정 작업(레이아웃 구성, 로그인 화면, 대시보드 틀, 다크모드 등)을 미리 완성해 둔 개발 출발점(템플릿)입니다. 개발자는 이 프로젝트를 복사해서 곧바로 비즈니스 기능 개발에 집중할 수 있습니다.

### 타깃 사용자

| 사용자 유형 | 설명 |
|-------------|------|
| 웹 서비스 개발자 | 새 프로젝트를 빠르게 시작하고 싶은 프론트엔드(화면 담당) 개발자 |
| 스타트업 팀 | 빠른 프로토타입(시제품) 제작이 필요한 소규모 팀 |
| 1인 개발자 | 혼자서 서비스 전체를 구축하는 개발자 |

### 핵심 가치

- **시간 절약**: 기본 화면 구조와 컴포넌트(화면 구성 요소)를 미리 제공하여 개발 초기 셋업 시간을 대폭 단축합니다.
- **일관된 품질**: 검증된 최신 기술 조합으로 처음부터 높은 품질의 코드 구조를 보장합니다.
- **확장 준비 완료**: 사용자 인증, 대시보드, API(외부 서비스 연결 통로) 연동 등 실제 서비스에 필요한 구조가 이미 갖춰져 있습니다.

## 2. 서비스 구조

### 페이지 목록 및 역할

| 경로 | 페이지명 | 역할 |
|------|----------|------|
| / | 홈(랜딩 페이지) | 서비스 소개 및 주요 기능 안내. 대시보드 및 로그인으로 이동 유도 |
| /login | 로그인 | 이메일과 비밀번호로 계정에 접속하는 화면 |
| /signup | 회원가입 | 새 계정을 만드는 화면 |
| /dashboard | 대시보드 개요 | 전체 사용자 수, 매출, 주문, 활성 접속자 등 핵심 지표 요약 |
| /dashboard/analytics | 분석 (예정) | 상세 통계 및 데이터 분석 화면 (사이드바에 메뉴 구성됨) |
| /dashboard/users | 사용자 관리 (예정) | 서비스 가입자 목록 및 관리 화면 |
| /dashboard/posts | 게시물 관리 (예정) | 콘텐츠 등록 및 관리 화면 |
| /dashboard/settings | 설정 (예정) | 서비스 환경 설정 화면 |
| /forgot-password | 비밀번호 찾기 (예정) | 비밀번호를 잊었을 때 재설정하는 화면 |
| /terms | 이용약관 (예정) | 서비스 이용 약관 안내 |
| /privacy | 개인정보처리방침 (예정) | 개인정보 처리 방침 안내 |
| /contact | 문의하기 (예정) | 고객 문의 접수 화면 |
| 존재하지 않는 주소 | 404 오류 페이지 | 잘못된 주소로 접근 시 안내 화면 |

> (예정)으로 표시된 페이지는 사이드바 메뉴 또는 링크에 경로가 구성되어 있으나 아직 화면 내용이 구현되지 않은 상태입니다.

### 레이아웃 계층 구조

모든 페이지는 공통 틀(Header + 본문 + Footer) 안에 들어갑니다. 대시보드(/dashboard 이하) 페이지에는 추가로 왼쪽 사이드바 메뉴가 함께 표시됩니다.

```
전체 화면 틀 (테마 관리 포함)
  ├── 상단 헤더 (로고, 네비게이션, 테마 전환 버튼)
  ├── 본문 영역
  │     ├── 홈, 로그인, 회원가입 페이지 (본문 전체 사용)
  │     └── 대시보드 페이지 (왼쪽 사이드바 + 오른쪽 콘텐츠 분할)
  ├── 하단 푸터 (저작권, 이용약관, 개인정보처리방침, 문의하기)
  └── 알림 메시지 표시 영역 (화면 전체에 겹쳐서 표시)
```

## 3. 주요 기능 명세

### 3-1. 홈 화면 (랜딩 페이지)

**목적**: 서비스 또는 스타터킷의 가치를 첫눈에 전달하고 사용자를 다음 행동(대시보드 보기 / 로그인)으로 유도합니다.

**구성 요소**:
- 히어로 섹션(화면 상단의 핵심 소개 영역): 큰 제목, 설명 문구, 두 개의 행동 유도 버튼(대시보드 보기, 로그인), 기술 스택 뱃지
- 기능 소개 카드 섹션: "빠른 개발", "TypeScript(코드 오류를 미리 잡아주는 언어)", "다크모드" 세 가지 특징을 카드 형태로 나열

**동작 흐름**:
1. 사용자가 홈 주소(/)에 접속합니다.
2. 화면 상단에 히어로 섹션이 표시됩니다.
3. "대시보드 보기" 버튼을 누르면 /dashboard로 이동합니다.
4. "로그인" 버튼을 누르면 /login으로 이동합니다.
5. 스크롤을 내리면 기능 소개 카드 3개가 표시됩니다.

---

### 3-2. 로그인

**목적**: 기존 계정을 가진 사용자가 서비스에 접속할 수 있게 합니다.

**구성 요소**:
- 이메일 입력칸
- 비밀번호 입력칸
- 비밀번호 찾기 링크
- 로그인 버튼
- 회원가입 페이지 이동 링크

**동작 흐름**:
1. 사용자가 /login에 접속하면 화면 중앙에 로그인 카드가 표시됩니다.
2. 이메일과 비밀번호를 입력합니다.
3. 비밀번호를 잊었다면 "비밀번호 찾기" 링크를 눌러 /forgot-password로 이동합니다.
4. "로그인" 버튼을 누르면 입력 정보가 서버로 전달됩니다.
5. 계정이 없는 사용자는 하단의 "회원가입" 링크를 눌러 /signup으로 이동합니다.

> 현재 상태: 화면 구조(UI)만 구현되어 있으며 실제 로그인 처리 로직(서버 통신, 세션 생성 등)은 연결되어 있지 않습니다. 실제 서비스 적용 시에는 NextAuth.js(인증 처리 도구) 등을 연동해야 합니다.

---

### 3-3. 회원가입

**목적**: 새 사용자가 계정을 만들 수 있게 합니다.

**구성 요소**:
- 이름 입력칸 (이름 / 성 두 칸으로 분리)
- 이메일 입력칸
- 비밀번호 입력칸 (8자 이상 안내 포함)
- 비밀번호 확인 입력칸
- 회원가입 버튼
- 로그인 페이지 이동 링크

**동작 흐름**:
1. 사용자가 /signup에 접속하면 화면 중앙에 회원가입 카드가 표시됩니다.
2. 이름, 성, 이메일, 비밀번호, 비밀번호 확인을 입력합니다.
3. "회원가입" 버튼을 누르면 입력 정보가 서버로 전달됩니다.
4. 이미 계정이 있는 사용자는 하단의 "로그인" 링크를 눌러 /login으로 이동합니다.

> 현재 상태: 화면 구조만 구현되어 있으며 실제 계정 생성 로직은 연결되어 있지 않습니다.

---

### 3-4. 대시보드

**목적**: 서비스 운영 현황을 한눈에 파악할 수 있는 관리자 전용 화면입니다.

**구성 요소**:
- 왼쪽 사이드바: 개요, 분석, 사용자, 게시물, 설정 메뉴
- 지표 요약 카드 4개: 전체 사용자 수, 월 매출, 활성 세션(현재 접속 중인 사용자), 전체 주문 수
- 최근 활동 목록: 최근 사용자들의 행동 기록 (계정 생성, 주문 완료, 비밀번호 변경 등)

**동작 흐름**:
1. 사용자가 /dashboard에 접속합니다.
2. 서버가 통계 데이터를 불러옵니다 (현재는 미리 입력된 예시 데이터 사용).
3. 지표 카드 4개가 상단에 격자(그리드) 형태로 표시됩니다.
4. 그 아래에 최근 활동 5건이 목록으로 표시됩니다.
5. 사이드바에서 다른 메뉴를 선택하면 해당 페이지로 이동합니다 (현재 해당 페이지 미구현).
6. 현재 선택된 메뉴 항목은 배경색이 강조되어 표시됩니다.

**지표 카드 상세**:

| 카드명 | 표시 내용 | 보조 설명 |
|--------|----------|-----------|
| 전체 사용자 | 1,248명 | 전월 대비 +12.5% |
| 월 매출 | ₩3,842,000 | 전월 대비 +8.3% |
| 활성 세션 | 72명 | 현재 접속 중인 사용자 |
| 전체 주문 | 384건 | 이번 달 처리된 주문 |

---

### 3-5. 다크모드 전환

**목적**: 사용자의 눈 피로를 줄이고 개인 선호에 맞는 화면 색상을 제공합니다.

**동작 흐름**:
1. 모든 페이지 상단 헤더 오른쪽에 테마 전환 버튼(태양 / 달 아이콘)이 표시됩니다.
2. 처음 접속 시 운영체제(Windows, macOS 등) 또는 브라우저의 기본 테마 설정을 자동으로 따릅니다.
3. 사용자가 버튼을 누르면 밝은 모드와 어두운 모드가 즉시 전환됩니다.
4. 현재 모드에 따라 태양 아이콘(밝은 모드) 또는 달 아이콘(어두운 모드)이 표시됩니다.
5. 선택한 테마는 다음 방문 시에도 유지됩니다.

---

### 3-6. 알림 메시지 (Toast)

**목적**: 사용자 행동에 대한 결과(성공, 오류, 경고, 정보)를 화면 한쪽에 잠깐 표시하는 알림 기능입니다.

**특징**:
- 현재 적용된 테마(밝음/어둠)에 맞춰 알림 색상도 자동으로 변경됩니다.
- 성공(체크 아이콘), 정보(i 아이콘), 경고(삼각형 아이콘), 오류(팔각형 X 아이콘), 로딩(회전 아이콘) 5가지 유형을 지원합니다.
- 알림은 화면 콘텐츠 위에 겹쳐서 표시되며, 일정 시간이 지나면 자동으로 사라집니다.

---

### 3-7. 404 오류 페이지

**목적**: 존재하지 않는 주소로 접근한 사용자에게 안내 메시지를 보여주고 홈으로 안내합니다.

**동작 흐름**:
1. 사용자가 존재하지 않는 주소로 접근합니다.
2. "404"라는 큰 숫자와 함께 "페이지를 찾을 수 없어요" 안내 메시지가 표시됩니다.
3. "홈으로 돌아가기" 버튼을 누르면 홈(/) 으로 이동합니다.

## 4. 컴포넌트 설계

### 공통 컴포넌트

| 컴포넌트명 | 위치 | 역할 |
|------------|------|------|
| ThemeProvider | 전체 감싸기 | 다크모드 상태를 전체 앱에 전달하는 최상위 관리자 |
| Header | 모든 페이지 상단 | 로고, 상단 네비게이션 메뉴, 테마 전환 버튼을 포함하는 고정 헤더 |
| Footer | 모든 페이지 하단 | 저작권 표시 및 이용약관, 개인정보처리방침, 문의하기 링크 |
| ThemeToggle | 헤더 내부 | 밝은 모드/어두운 모드를 전환하는 버튼 |
| NavItem | 대시보드 사이드바 | 현재 위치에 따라 강조 표시가 달라지는 사이드바 메뉴 항목 |

### UI 구성 요소 (재사용 가능한 화면 블록)

| 컴포넌트명 | 역할 | 사용처 |
|------------|------|--------|
| Button | 다양한 모양과 크기의 버튼 | 로그인, 회원가입, CTA(행동 유도) 버튼 등 전반 |
| Card | 정보를 담는 카드 박스 | 홈 기능 소개, 로그인/회원가입 폼, 대시보드 지표 |
| Input | 텍스트 입력칸 | 로그인/회원가입 폼 |
| DropdownMenu | 클릭하면 펼쳐지는 선택 메뉴 | 향후 사용자 메뉴, 필터 등에 활용 가능 |
| Toaster | 화면 위에 떠오르는 알림 메시지 | 전체 앱에서 성공/오류/경고 알림 표시 |

### 컴포넌트 관계도

```
RootLayout
  └── ThemeProvider (다크모드 상태 관리)
        ├── Header
        │     ├── 로고 링크
        │     ├── 네비게이션 링크
        │     └── ThemeToggle
        ├── 페이지 본문
        │     ├── HomePage: 히어로 섹션 + 기능 카드
        │     ├── LoginPage: Card + Input + Button
        │     ├── SignupPage: Card + Input + Button
        │     ├── DashboardLayout
        │     │     ├── 사이드바 (NavItem 목록)
        │     │     └── DashboardPage: 지표 Card + 활동 목록
        │     └── NotFoundPage: 404 안내 + 홈 이동 버튼
        ├── Footer
        └── Toaster (알림 메시지)
```

## 5. 데이터 모델

### 사용자 (User)

서비스의 회원 정보를 나타내는 구조입니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| id | 고유 식별 번호 | "user-001" |
| name | 사용자 이름 | "홍길동" |
| email | 이메일 주소 | "hong@example.com" |
| role | 계정 권한 | "admin"(관리자) 또는 "user"(일반 사용자) |
| createdAt | 계정 생성 일시 | 2026-01-15 |

### API 응답 형식 (ApiResponse)

서버에서 데이터를 받아올 때 일관된 형식을 사용합니다.

| 항목 | 설명 |
|------|------|
| data | 실제 응답 데이터 |
| error | 오류 발생 시 오류 메시지 (정상이면 없음) |
| status | 처리 결과 코드 (200: 성공, 400: 요청 오류, 500: 서버 오류 등) |

### 목록 응답 형식 (PaginatedResponse)

목록 데이터를 여러 페이지로 나눠서 받아올 때 사용합니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| items | 현재 페이지의 항목 목록 | 사용자 20명 |
| total | 전체 항목 수 | 248 |
| page | 현재 페이지 번호 | 1 |
| pageSize | 한 페이지에 표시할 항목 수 | 20 |
| totalPages | 전체 페이지 수 | 13 |

### 테마 모드 (ThemeMode)

다크모드 설정값 유형입니다.

- "light": 밝은 화면
- "dark": 어두운 화면
- "system": 기기 설정 따름

### 대시보드 통계 (현재는 예시 데이터)

## 6. UI/UX 설계 원칙

### 색상 테마

흰색 계열의 무채색(neutral) 기반 디자인을 사용합니다. 화려한 강조색 없이 깔끔하고 전문적인 인상을 줍니다.

| 용도 | 밝은 모드 | 어두운 모드 |
|------|----------|------------|
| 배경 | 흰색 | 매우 짙은 어두운 색 |
| 주요 텍스트 | 거의 검정 | 거의 흰색 |
| 강조색(primary) | 짙은 회색 | 밝은 회색 |
| 카드 배경 | 흰색 | 짙은 어두운 회색 |
| 경계선 | 연한 회색 | 반투명 흰색(10%) |
| 위험 표시(오류) | 붉은색 계열 | 밝은 붉은색 계열 |

> oklch(색의 밝기, 채도, 색상을 수학적으로 표현하는 현대적인 색상 표기 방식)를 사용하여 밝은/어두운 모드 모두에서 색상이 일관되게 보입니다.

### 레이아웃 전략

- **최대 너비 제한**: 콘텐츠는 최대 1,120px까지만 넓어지며, 그보다 넓은 화면에서는 가운데 정렬됩니다. (너무 넓어져서 읽기 불편해지는 것을 방지)
- **고정 헤더**: 스크롤해도 항상 화면 상단에 헤더가 고정됩니다. 반투명 배경 효과(흐림 효과)로 콘텐츠와 구분됩니다.
- **유연한 본문**: 헤더와 푸터를 제외한 본문 영역이 남은 화면을 모두 채웁니다.
- **대시보드 2단 레이아웃**: 왼쪽 사이드바(224px 고정)와 오른쪽 콘텐츠 영역으로 분리됩니다.

### 반응형 전략

- 모바일 화면에서는 격자(그리드) 배열이 1열로 축소됩니다.
- 대시보드 지표 카드는 화면 크기에 따라 2열(중간 화면) 또는 4열(넓은 화면)로 자동 변경됩니다.
- 푸터 링크는 작은 화면에서 세로로 쌓이고, 큰 화면에서 가로로 펼쳐집니다.

### 폰트

- 일반 텍스트: Geist Sans (가독성이 높은 현대적 서체)
- 코드/숫자: Geist Mono (등폭 서체로 코드나 수치 정보 표현)
- 두 폰트 모두 구글 폰트에서 자동으로 불러옵니다.

### 모서리 둥글기 (Radius)

기본값 0.625rem을 기준으로 다양한 크기를 비례하여 적용합니다. 전체 UI에 일관된 곡선감을 줍니다.

### 인터랙션 원칙

- 버튼, 링크에 마우스를 올리면(hover) 색상이 부드럽게 전환됩니다.
- 키보드로 탐색할 때 현재 선택된 요소 주변에 테두리 링(ring)이 표시됩니다. (접근성 고려)
- 테마 전환 시 깜빡임(화면 전환 애니메이션)을 억제하여 자연스럽게 전환됩니다.
- 카드에 마우스를 올리면 그림자가 부드럽게 생깁니다.

## 7. 기술적 결정 사항

### 왜 이 기술들을 선택했는가?

**Next.js 16 (웹사이트를 빠르게 만들어주는 개발 도구) + App Router**

페이지 구조를 폴더 기반으로 관리합니다. 서버에서 화면을 그려 사용자에게 전달하는 방식(서버 사이드 렌더링)과 브라우저에서 동적으로 작동하는 방식을 유연하게 혼용할 수 있습니다. Turbopack(빠른 개발 빌드 도구)이 기본으로 활성화되어 개발 중 화면 갱신 속도가 매우 빠릅니다.

**"use cache" 지시자 (서버 데이터 자동 저장 기능)**

대시보드 통계 데이터를 서버에서 가져올 때 결과를 자동으로 저장(캐싱)합니다. 매번 데이터베이스나 외부 서비스에 요청하지 않고 저장된 결과를 재활용하여 응답 속도를 높입니다. Next.js 16의 새로운 기능으로, next.config.ts에서 cacheComponents: true로 활성화되어 있습니다.

**shadcn/ui (미리 만들어진 화면 구성 요소 모음) — base-nova 스타일**

컴포넌트 코드를 프로젝트 안으로 직접 가져오는 방식이라 자유롭게 수정할 수 있습니다. 이 프로젝트는 최신 스타일인 base-nova를 사용하며, 기존 Radix UI 대신 @base-ui/react라는 더 가볍고 현대적인 기반 라이브러리를 사용합니다.

**Tailwind CSS v4 (미리 만들어진 디자인 조각을 조합하는 스타일 도구)**

별도의 설정 파일(tailwind.config.ts) 없이 globals.css 하나에서 모든 디자인 토큰(색상, 간격, 폰트)을 관리합니다. v4의 @theme 블록을 활용하여 CSS 변수와 Tailwind 클래스가 서로 연결됩니다.

**next-themes (다크모드 관리 도구)**

시스템 설정 감지, 테마 저장, 깜빡임 없는 전환을 한 번에 해결해줍니다. suppressHydrationWarning 설정으로 서버와 브라우저 간 렌더링 불일치 경고를 방지합니다.

**sonner (알림 메시지 도구)**

가볍고 현대적인 알림 메시지 라이브러리입니다. 테마와 연동되어 밝은/어두운 모드에서 모두 적절한 색상으로 알림이 표시됩니다.

**TypeScript (코드 실수를 미리 잡아주는 엄격한 프로그래밍 언어)**

코드를 작성하는 시점에 타입 불일치 오류를 발견할 수 있어 런타임(실제 실행 중) 오류를 줄입니다. types/index.ts에 공통 타입을 모아 관리합니다.

**@base-ui/react (기능만 있고 스타일은 없는 UI 기반 라이브러리)**

버튼, 입력창, 드롭다운 메뉴 등의 클릭, 키보드 탐색, 접근성 기능을 기본으로 제공하면서 디자인은 완전히 자유롭게 적용할 수 있습니다. Radix UI보다 더 최신 설계 철학을 따릅니다.

**class-variance-authority (CVA, 컴포넌트 변형 관리 도구)**

하나의 버튼 컴포넌트에서 variant(default/outline/ghost 등)와 size(sm/default/lg 등)를 조합하여 다양한 모양을 만들어냅니다. 복잡한 조건부 클래스 처리를 깔끔하게 관리합니다.

## 9. 향후 개선 방향

현재 구현 상태를 기반으로 파악된 다음 단계 과제들입니다.

### 즉시 필요한 구현

1. **인증 기능 연결**: 로그인, 회원가입 폼에 실제 서버 통신 로직(NextAuth.js 또는 직접 구현) 연결이 필요합니다.
2. **미구현 페이지 완성**: 분석(/dashboard/analytics), 사용자 관리(/dashboard/users), 게시물 관리(/dashboard/posts), 설정(/dashboard/settings), 비밀번호 찾기(/forgot-password) 페이지 구현이 필요합니다.
3. **법적 고지 페이지**: 이용약관(/terms), 개인정보처리방침(/privacy) 페이지를 실제 내용으로 채워야 합니다.
4. **문의 폼**: /contact 페이지에 실제 문의를 접수할 수 있는 폼이 필요합니다.

### 기능 확장 가능성

5. **소셜 로그인**: 환경 변수 파일(.env.example)에 GitHub, Google OAuth(소셜 계정으로 로그인하는 방식) 설정 항목이 미리 준비되어 있어 손쉽게 추가할 수 있습니다.
6. **데이터베이스 연결**: 환경 변수에 DATABASE_URL(데이터베이스 연결 주소) 항목이 준비되어 있어 PostgreSQL(데이터 저장 도구) 연결이 용이합니다.
7. **권한 기반 접근 제어**: User 타입에 role 필드("admin"/"user")가 이미 정의되어 있어 관리자/일반 사용자 구분 기능을 추가할 준비가 되어 있습니다.
8. **목록 페이지네이션(페이지 나누기)**: PaginatedResponse 타입이 이미 정의되어 있어 사용자 목록, 게시물 목록 등에 바로 적용할 수 있습니다.
9. **드롭다운 메뉴 활용**: DropdownMenu 컴포넌트가 구현되어 있으나 아직 실제로 사용되는 곳이 없습니다. 헤더의 사용자 프로필 메뉴, 대시보드의 필터 메뉴 등에 활용할 수 있습니다.
10. **모바일 사이드바 개선**: 현재 대시보드 사이드바는 작은 화면에서 항상 표시됩니다. 모바일에서는 숨겼다가 햄버거 메뉴 버튼으로 열 수 있도록 개선이 필요합니다.
11. **알림 메시지 활용**: Toaster 컴포넌트가 루트 레이아웃에 포함되어 있지만 현재 실제로 알림을 발생시키는 코드가 없습니다. 로그인 성공/실패, 데이터 저장 완료 등의 피드백 메시지를 추가할 수 있습니다.
12. **차트/그래프 추가**: 대시보드에 차트 색상 변수(--chart-1 ~ --chart-5)가 미리 정의되어 있어 그래프 라이브러리(Chart.js 또는 Recharts 등)를 쉽게 연동할 수 있습니다.
